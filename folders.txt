● .planning 文件夹是由 GSD (Get Stuff Done) 技能集创建和管理的。

  GSD 技能集

  GSD 是一组结构化的项目管理技能，以 gsd: 为前缀：

  ┌───────────────────┬────────────────────────────────────────┐
  │       技能        │                  用途                  │
  ├───────────────────┼────────────────────────────────────────┤
  │ gsd:new-project   │ 初始化新项目，创建 .planning/ 结构     │
  ├───────────────────┼────────────────────────────────────────┤
  │ gsd:plan-phase    │ 创建阶段计划 (PLAN.md)                 │
  ├───────────────────┼────────────────────────────────────────┤
  │ gsd:execute-phase │ 执行阶段计划                           │
  ├───────────────────┼────────────────────────────────────────┤
  │ gsd:map-codebase  │ 代码库分析，输出到 .planning/codebase/ │
  ├───────────────────┼────────────────────────────────────────┤
  │ gsd:discuss-phase │ 阶段讨论，收集上下文                   │
  └───────────────────┴────────────────────────────────────────┘

  .planning 文件夹作用

  .planning/
  ├── research/           # 研究文档
  │   └── RESEARCH.md
  ├── codebase/           # 代码库分析
  │   ├── tech-analysis.md
  │   └── arch-analysis.md
  ├── phase-*/            # 阶段目录
  │   ├── PLAN.md
  │   ├── VERIFICATION.md
  │   └── CHECKPOINT.md
  ├── PROJECT.md          # 项目总览
  └── ROADMAP.md          # 路线图

  主要用途：
  1. 项目规划 - 存储项目结构、路线图、需求文档
  2. 阶段管理 - 每个开发阶段有独立的计划和验证文档
  3. 研究记录 - 技术调研、架构决策记录
  4. 代码库映射 - 自动分析的代码库结构文档
  5. 工作流状态 - 跟踪当前进度和下一步行动
