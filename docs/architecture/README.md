### 架构原则

1. **分层架构**: Web → Application → Domain → Infrastructure
2. **依赖倒置**: 内层不依赖外层，通过接口解耦
3. **关注点分离**: 每个组件职责单一
4. **安全优先**: 纵深防御，多层安全机制
5. **可测试性**: 依赖注入，便于单元测试和集成测试

### 关键指标

| 指标 | 目标 | 说明 |
|------|------|------|
| 测试覆盖率 | ≥ 85% | JaCoCo 报告 |
| API 响应时间 | P95 < 200ms | 数据库优化 + 缓存 |
| 并发用户 | 50,000 峰值 | 虚拟线程 + 水平扩展 |
| 安全审计 | 100% 覆盖 | 所有敏感操作记录 |
